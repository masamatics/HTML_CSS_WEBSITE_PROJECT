Lesson 1

The World Wide Web

The web is composed largely of HTML documents. HTML stands for Hypertext Markup Languageand is the language the browsers use to interpret web pages. The components of the web include the client (your computer and browser), the Internet, and servers. The client and servers interact through something called the Hypertext Transfer Protocol (HTTP).

Basic HTML Elements

HTML Elements are what we use to tell a web browser how to display content in a web page. Most HTML elements consist of an Opening and a Closing tag with some content between them.

For example:
<b>text</b> makes the 'text' content bold
<em>text</em> makes the 'text' content italic

HTML Attributes

Some elements accept additional values called attributes in their opening tags. Attributes provide additional information to the browser. For example, the anchor tag takes an attribute called href that defines a URL to link to.

<a href="url">link</a> creates a hyperlink to the href attribute value url and names it link

<img src="url" alt="text"> displays the image hosted at the src attribute value "url" (this can also be a path to a local file on your computer). If the image cannot be displayed, then the alt attribute value "text" is shown.

More HTML Elements

  • <p>text</p> defines a paragraph. The content of the <p> element is in a "box."
  • <br> creates a line break.
  • <div>content</div> creates a "box" container element.
  • <span>content</span> creates an "inline" container element.

For more information, take a look at Tree structure on Wikipedia

HTML-CSS-DOM

  • HTML - HyperText Markup Language - has specific Syntax and Rule. Generally describes content between tags.
  • CSS - Cascading Style Sheets - adds style to the contents that were described in HTML.
  • DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.
Notes:
  • HTML and CSS are both "languages"
  • HTML controls the "structure" of a web page
  • CSS controls the "style" of a page.
  • When programmers talk about the "DOM" (Document Object Model) they are talking about the tree-like structure of a page.

Boxes, Grids and Rules in HTML

Everything on the webpage is boxes. Thinking of things in terms of boxes on the webpage is a lot easier as web developer. For example, you can rearrange positioning of the contents in the webpages.

When you are making webpage, in HTML each box is represented by div tab: <div>.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text Editors (for programming)

When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3

Adding Style

In the HTML document, you need to link to CSS document, which can be done as in highlights below.

<!DOCTYPE html>
<html>
<head>
         <link rel = "stylesheet" type="text/css" href="style.css">
</head>

In this case, the HTML document is linking the CSS document named style.css. The CSS document must be saved in the same location as HTML document in order for browser to link them together.

Understanding CSS

CSS is...

As a Search and Replace tool. You can identify and find ("match" in CSS terminology) a class or a tag of the element

Also itan be defined in different ways such as following;

- the default style of a browser (different browsers have slightly different styles)

- stylesheet in a separate file (this is what you will be mostly using)

<head>
  <link rel="stylesheet" href="main.css">
</head>
<body>
  <div>
     This will have a red background.
  </div>
</body>

-----(on a separate .css file)---------
div {
    background-color : red;
}

- stylesheet inside HTML (this can be done for small projects but is not ideal)

<head>
  <style>
  div {
     background-color : red;
  }
  </style>
</head>

- inline style in an element (this can also be done but should be avoided)

<div style="background-color: red; color: white">
This div will have a red background and white text.

"Cascading" means that rules are applied to all of those element's child elements. However, if multiple rules are set to child elements, the more specific rule takes effect.

However, Sometimes these rules are treated differently depending on the kind of browsers (Google Chrome, Safari, Firefox, Internet Explorer).
Normalize.css precisely targets the style that need normalizing and makes browsers render all elements more consistently and in line with modern standards.

How to write CSS

It looks like this

h1 {
  background-color : black;
  color: white;
}

Simply put the selector / id attribute in HTML, followed by braces {}, which contains style elements inside. In the example above, selector, h1 has a rule defined as background color of black and text color of white.

CSS Vocabularies:
CSS Cascading Style Sheets
Style This word can refer to many things and so it can be confusing. It can refer to:
  • The HTML element. For example: <style> div {color:blue} </style>
  • The HTML attribute. For example: <div> style="color : blue">this text would be blue</div>
  • The general look of a web page. For example: "I like that site's style."
  • A verb. For example: "I'm making progress on my page. The structure is all done but now I have to style it."
Rule a line of CSS code describing the value that a certain attribute should take.
Property The property you want to change
Value The value that you want to assign to the attribute
Selector The name that you use to in order to target the elements that are assigned to a class or id attribute in the HTML
Class A class refers to a group of elements that can be styled together. Class names should not contain periods or any other punctuation marks such as class="1.1"
ID ID's are unique identifiers that uniquely identifies an element in HTML

The Boxes

Understanding the box model is important in order to build a website.
Content is wrapped by padding, border and margin. You can change these in developer tools in the browser.

  • Padding is inside the box to wrap content
  • Border is the line of the box
  • Margin is the space between boxes

In order to set box-sizing definition and outline that allows you to easily see all the boxes on your page, put the below code in the css file.

* {
  outline: 1px solid red !important;
}
* {
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box;
    -ms-box-sizing: border-box;
    box-sizing: border-box;
}

Positioning the Boxes

Flexbox (Flexible box) - you can let divs appear next to each other by adding below in .css file

.app {
    display: -webkit-flex;
    display: flex;
}
